\section{Experimental Setup}
\label{sec:experimental-setup}

We evaluate Semantic Fact DB against two baselines:
KnowledgeGraph (our RDF-inspired triple store baseline) and
SheafDatabase (the sheaf-based representation).
Apache Jena TDB2 integration is in progress; the adapter
scaffold is in \texttt{src/sfdb/benchmark/engine\_adapter.py}.

\subsection{Hardware}
All experiments run on a single machine with an
\texttt{\showcpu}
processor and
\texttt{\showmemory}
of RAM, running
\texttt{\showos}.

\subsection{Implementation}
Semantic Fact DB is implemented in Python 3.14.
Both the KG and Sheaf backends share the same
parser, logical plan representation, and canonical model.

\subsection{Datasets}
We use synthetic datasets with controlled size, arity, and context depth,
generated with seed 42 for reproducibility.
LUBM integration is in progress; real-world dataset
evaluation is left as future work.

\subsection{Query Workloads}
We evaluate two query types: LOOKUP (entity-centric retrieval via
subject identifier) and GLOBAL (full scan of all facts).
These correspond to the C1--C10 workload taxonomy described in
Section~\ref{sec:contextual-workloads}.

\subsection{Metrics}
For each query we measure latency (ms).
All metrics report the mean over 10 warm-cache runs
after 2 warm-up iterations.
