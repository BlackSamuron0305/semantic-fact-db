\section{Problem Statement}
\label{sec:problem}

We formalise the storage and query problem that both engines are required to
solve. The formalism is deliberately compact: it names only the objects that
appear in the implementation and in the equivalence proof.

\subsection{Informal Statement}
\label{sec:problem:informal}

The system is given a finite stream of semantic facts. Each fact is a tuple
of arguments together with a context of validity. The system must accept
inserts, answer queries that either fix a context or range freely, and expose
a canonical view against which any concrete storage engine can be checked for
semantic equivalence.

\subsection{Formal Statement}
\label{sec:problem:formal}

Let $\mathcal{F}$ denote the set of well-formed semantic facts, with fields
$(\mathrm{id}, s, r, \vec{o}, c, k, m)$ as defined in
Definition~\ref{def:semanticfact}. Let $(\mathcal{C}, \leq)$ be the finite
context poset defined in Definition~\ref{def:context}, and let $\mathcal{Q}$
be the query language of Section~\ref{sec:arch:query}. A \emph{storage
engine} is a data structure $\Sigma$ that maintains an internal representation
of a finite set $F \subseteq \mathcal{F}$ and exposes the following
operations:

\begin{description}
    \item[$\textsc{Insert}(f)$] adds $f$ to $F$.
    \item[$\textsc{Query}(Q)$] returns the set of facts in $F$ that match
          $Q$, under the denotational semantics of Section~\ref{sec:arch:query}.
    \item[$\textsc{Glue}()$] returns the global sections of $F$: the maximal
          collection of facts whose local sections are pairwise compatible
          on their common context overlaps.
\end{description}

Two storage engines $\Sigma_1$ and $\Sigma_2$ over the same $F$ are
\emph{semantically equivalent} if
\[
    \llbracket Q \rrbracket_{\Sigma_1} \;\cong\; \llbracket Q \rrbracket_{\Sigma_2}
    \qquad \text{for every } Q \in \mathcal{Q},
\]
where $\cong$ denotes equality of the associated canonical result sets in
the sense of Definition~\ref{def:semanticequiv}.

\subsection{What We Actually Prove and Measure}
\label{sec:problem:scope}

The equivalence claim above is proved constructively in
Section~\ref{sec:correctness} through a canonical intermediate representation
and checked at every insert and every query by the shared verification
harness. The performance claim---that a context-indexed engine can answer
entity-centric lookups, unrestricted scans, and bounded temporal-range
queries faster than a reified triple store by trading resident memory for
latency---is measured empirically in Section~\ref{sec:evaluation}.
