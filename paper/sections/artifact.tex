\subsection{Reproducing the Results}
\label{sec:artifact:reproducibility}

Every benchmark run records a reproducibility manifest that identifies the
git commit, Python interpreter version, \texttt{uv.lock} hash, dataset
checksum, and random seed used to produce it. Table~\ref{tab:reproducibility}
lists the values used for the run reported in this paper.

\begin{table}[ht]
\centering
\caption{Reproducibility metadata for the reported run.}
\label{tab:reproducibility}
\begin{tabular}{@{}ll@{}}
\toprule
Property & Value \\
\midrule
Git commit & \texttt{\githash} \\
Python version & \texttt{\pythonversion} \\
uv lock hash & \texttt{\uvlockhash} \\
Dataset checksum & \texttt{\datachecksum} \\
Random seed & \texttt{\sfdbseed} \\
Generation date & \texttt{\generationdate} \\
\bottomrule
\end{tabular}
\end{table}

To reproduce the reported numbers from a clean clone:

\begin{verbatim}
git clone https://github.com/BlackSamuron0305/semantic-fact-db
cd semantic-fact-db
uv sync --group dev
uv run pytest                          # 358 tests should pass
uv run sfdb benchmark                  # all 3 scales, populates results/
uv run python scripts/generate_tables.py  # regenerates paper/tables/*.tex
uv run python scripts/generate_paper.py
cd paper && make                       # produces main.pdf
\end{verbatim}

The \texttt{sfdb benchmark} command with no \texttt{--size} argument runs
the full paper suite (100, 1\,000, and 10\,000 facts; 10 timed runs and 2
warm-up iterations per query class, per the methodology in
Section~\ref{sec:eval:setup}). It writes raw per-run measurements to
\texttt{results/paper\_suite.csv} (also as JSON and Markdown) and an
aggregated \texttt{results/paper\_suite\_summary.json}. Every query class
at every scale is cross-engine verified as part of the same run; the
command exits non-zero if any verification fails.
