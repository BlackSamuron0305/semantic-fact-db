# Formal Definitions

> All notation follows the conventions established in `notation.md`.

---

## Definition 1: SemanticFact

A **semantic fact** is a structured proposition represented as a tuple:

\[
f = (i, s, r, \vec{o}, c, k, m)
\]

where:

| Component | Symbol | Type | Meaning |
|-----------|--------|------|---------|
| Identifier | $i$ | $\mathcal{I}$ | Globally unique UUID identifying this fact |
| Subject | $s$ | $\mathcal{E} \subseteq \mathcal{I}$ | The entity the fact is about |
| Relation | $r$ | $\mathcal{R} \subseteq \mathcal{I}$ | The type of relationship |
| Objects | $\vec{o}$ | $\mathcal{V}^*$ | Ordered sequence of values (literals or entity references) |
| Context | $c$ | $\mathcal{C}$ | The context in which this fact holds |
| Confidence | $k$ | $[0,1] \subset \mathbb{R}$ | Degree of belief in the fact |
| Metadata | $m$ | $\mathbf{Set}$ | Arbitrary additional structured data |

Let $\mathcal{F}$ denote the set of all possible semantic facts.

**Invariant:** For any fact $f = (i, s, r, \vec{o}, c, k, m)$:
- $i$ is unique across all facts in a knowledge state.
- $k \in [0, 1]$.
- $\vec{o}$ has length $\operatorname{arity}(f) \geq 0$, where $\operatorname{arity}(f) = |\vec{o}|$.

**Implementation:** See `sfdb.common.types.Fact` and `common.schema.SemanticFact`.

## Definition 2: Entity

An **entity** is a distinguished identifier denoting a real-world object, person, event, or abstract concept:

\[
\varepsilon \in \mathcal{E} \subseteq \mathcal{I}
\]

Every entity $\varepsilon$ has:
- A unique identifier $\operatorname{id}(\varepsilon) \in \mathcal{I}$.
- A semantic type $\operatorname{type}(\varepsilon) \in \mathbf{SemanticType}$.
- An optional name $\operatorname{name}(\varepsilon)$.
- An optional set of attributes $\operatorname{attrs}(\varepsilon) \subseteq \mathcal{V}^*$.

**Entities as subjects:** If $f = (i, s, r, \vec{o}, c, k, m)$ is a fact, then $s$ is necessarily an entity: $s \in \mathcal{E}$.

**Entities as objects:** An object slot $\omega_j \in \vec{o}$ is an entity reference if $\omega_j$ has type $\mathtt{Reference}$; otherwise it is a literal.

**Implementation:** See `sfdb.canonical.canonical.CanonicalEntity`.

---

## Definition 3: Relation

A **relation** (role, predicate) is a distinguished identifier that types the connection between a subject and its objects:

\[
\rho \in \mathcal{R} \subseteq \mathcal{I}
\]

Every relation $\rho$ has:
- A unique identifier $\operatorname{id}(\rho) \in \mathcal{I}$.
- An arity $\operatorname{arity}(\rho) \geq 0$ (number of expected object slots).
- A sequence of slot types $\operatorname{slots}(\rho) \in \mathbf{SemanticType}^*$ of length $\operatorname{arity}(\rho)$.

**Relation application:** A fact $f = (i, s, r, \vec{o}, c, k, m)$ is well-typed iff $\operatorname{arity}(r) = |\vec{o}|$ and for each $j$, $\omega_j$ conforms to $\operatorname{slots}(r)_j$.

**Implementation:** See `sfdb.canonical.canonical.CanonicalRelation`.

## Definition 4: Context

A **context** is a node in a finite poset $(\mathcal{C}, \leq)$ representing a scope of discourse.

Each context $c \in \mathcal{C}$ is identified by a dot-separated path:

\[
c = \text{``root''}.\text{``domain''}.\text{``subdomain''}\ldots
\]

The partial order $\leq$ is the prefix order: $c_1 \leq c_2$ iff $c_2$ is a prefix of $c_1$, i.e., $c_1$ is a **sub-context** (more specific) of the broader context $c_2$.

**Properties:**
- Reflexivity: $c \leq c$ for all $c \in \mathcal{C}$.
- Antisymmetry: $c_1 \leq c_2$ and $c_2 \leq c_1$ implies $c_1 = c_2$.
- Transitivity: $c_1 \leq c_2$ and $c_2 \leq c_3$ implies $c_1 \leq c_3$.

**Root context:** There exists a unique top element $\top \in \mathcal{C}$ (the root, typically "world") such that $c \leq \top$ for all $c \in \mathcal{C}$.

**Depth:** $\operatorname{depth}(c) = n$ where $c$ has $n$ dot-separated segments. The root has depth $0$.

**Meet:** The meet (greatest lower bound) $c_1 \wedge c_2$ is the longest common prefix of $c_1$ and $c_2$, which is the most specific context that is a sub-context of both.

**Implementation:** See `sfdb.common.types.Context`.

## Definition 5: Finite Topological Space

A **finite topological space** is a pair $(X, \mathcal{T})$ where:
- $X$ is a finite set of **points**.
- $\mathcal{T} \subseteq \mathcal{P}(X)$ is a collection of **open sets** satisfying:
  1. $\emptyset \in \mathcal{T}$ and $X \in \mathcal{T}$.
  2. If $U, V \in \mathcal{T}$, then $U \cap V \in \mathcal{T}$ (finite intersection closure).
  3. If $\{U_i\}_{i \in I} \subseteq \mathcal{T}$ for any index set $I$, then $\bigcup_{i \in I} U_i \in \mathcal{T}$ (arbitrary union closure).

For the SheafDB:
- $X = \mathcal{I}$ (the set of fact identifiers).
- $\mathcal{T}$ is generated by assigning each fact $f$ with context $c$ to the open set $\downarrow c \subseteq \mathcal{C}$, then pulling back along an assignment map $\alpha: \mathcal{I} \to \mathcal{C}$.

**Implementation:** See `sfdb.sheaf.topology.FiniteTopologicalSpace`.

---

## Definition 6: Open Set

An **open set** $U \in \mathcal{T}$ is a subset of $X$ that is declared open by the topology.

In the Alexandrov topology induced by $(\mathcal{C}, \leq)$:
- Every down-set $\downarrow c = \{d \in \mathcal{C} \mid d \leq c\}$ is open.
- The full topology $\mathcal{T}_A$ consists of all down-sets of $\mathcal{C}$.

**Open set construction:** To each context $c \in \mathcal{C}$, we associate the open set $U_c = \{f \in \mathcal{I} \mid \operatorname{context}(f) \leq c\}$, i.e., the set of all fact identifiers whose context is a sub-context of $c$.

**Properties:**
- $U_{\top} = X$ (the whole space).
- $U_{c_1} \cap U_{c_2} = U_{c_1 \wedge c_2}$.
- $U_{c_1} \cup U_{c_2} = U_{c_1 \vee c_2}$ when the join exists.

**Implementation:** See `sfdb.sheaf.topology.OpenSet`.

---

## Definition 7: Neighbourhood

A **neighbourhood** of a point $x \in X$ is any open set $U \in \mathcal{T}$ such that $x \in U$.

The **neighbourhood filter** of $x$ is:

\[
\mathcal{N}(x) = \{U \in \mathcal{T} \mid x \in U\}.
\]

In the Alexandrov topology, the **minimal neighbourhood** of $x$ exists uniquely:

\[
\mathcal{B}(x) = \bigcap_{U \in \mathcal{N}(x)} U = \downarrow \alpha(x),
\]

where $\alpha: X \to \mathcal{C}$ maps each point to its context. This is the smallest open set containing $x$.

**Neighbourhood expansion:** Given a point $x$, the set of neighbourhoods ordered by reverse inclusion forms a directed set:

\[
\mathcal{B}(x) \supseteq \mathcal{B}(x) \cup \mathcal{B}(y_1) \supseteq \cdots
\]

**Implementation:** See `sfdb.sheaf.topology.Neighborhood`, `FiniteTopologicalSpace.neighborhoods()`.

## Definition 8: Restriction Map

Let $c, d \in \mathcal{C}$ with $c \leq d$ (so $c$ is a sub-context of $d$). The **restriction map**

\[
\rho_{d,c}: \mathcal{P}(\mathcal{F}_d) \to \mathcal{P}(\mathcal{F}_c)
\]

sends a set of facts valid in context $d$ to the corresponding set valid in the more specific context $c$.

Restriction acts on individual facts as:

\[
\rho_{d,c}(f) = (i, s, r, \vec{o}', c, k, m)
\]

where $\vec{o}'$ is obtained from $\vec{o}$ by:
1. Dropping object slots that are not meaningful in $c$.
2. Specialising values according to the more specific context.

**Functoriality:** Restriction maps satisfy:
1. $\rho_{c,c} = \operatorname{id}_{\mathcal{P}(\mathcal{F}_c)}$ (identity).
2. $\rho_{e,c} = \rho_{d,c} \circ \rho_{e,d}$ for $c \leq d \leq e$ (composition).

**Implementation:** See `sfdb.common.types.RestrictionMap`, `sfdb.sheaf.sheaf.Presheaf.restrict()`.

## Definition 9: Stalk

Let $F$ be a presheaf on $(X, \mathcal{T})$. The **stalk** of $F$ at a point $x \in X$ is the set of germs defined as the direct limit:

\[
F_x = \varinjlim_{U \in \mathcal{N}(x)} F(U).
\]

Explicitly, $F_x$ is the set of equivalence classes:

\[
F_x = \left\{[s]_U \;\middle|\; U \in \mathcal{N}(x),\; s \in F(U)\right\} / \sim
\]

where $[s]_U \sim [t]_V$ iff there exists $W \subseteq U \cap V$ with $x \in W$ such that $s|_W = t|_W$.

**Interpretation for SheafDB:** The stalk $F_f$ at fact identifier $f$ contains all information about $f$ that is visible in arbitrarily small neighbourhoods of $f$. It represents the "germ" of the fact — its essential content independent of the specific choice of open set.

## Definition 10: Local Section

A **local section** is a section $s \in F(U)$ defined over a proper open subset $U \subsetneq X$.

In SheafDB terms, a local section is a fact $f$ whose context $c$ is a proper sub-context of the root: $c < \top$. The fact is valid in $c$ but may not be valid in broader contexts without gluing.

**Properties:**
- Every local section can be restricted to any smaller open set $V \subseteq U$.
- Local sections may be **compatible** (agree on overlaps) or **incompatible** (disagree).

**Implementation:** All facts stored via `SheafStore.insert()` are initially local sections assigned to their given context.

---

## Definition 11: Global Section

A **global section** is a section $s \in F(X)$ defined over the whole space $X$.

In SheafDB terms, a global section is a fact that is valid in the root context $\top$ ("world"). It holds everywhere.

The set of global sections $\Gamma(X, F)$ is computed by:
1. Collecting all local sections.
2. Verifying compatibility on pairwise overlaps (meets of contexts).
3. Gluing compatible families into sections over successively broader contexts.
4. Repeating until the root context is reached.

**Uniqueness:** If a gluing exists, the resulting global section is unique (sheaf axiom).

**Implementation:** See `sfdb.sheaf.sheaf.Sheaf.global_sections()`.

## Definition 12: Knowledge State

A **knowledge state** is a pair $\Sigma = (X, F)$ where:
- $X$ is a finite topological space of fact identifiers.
- $F$ is a presheaf (or sheaf) on $X$ assigning to each open set $U \subseteq X$ the set of facts valid in $U$.

Equivalently, $\Sigma$ is a set of semantic facts $\{f_1, \ldots, f_n\}$ together with a context assignment $\alpha: \mathcal{I} \to \mathcal{C}$, equipped with the Alexandrov topology generated by $(\mathcal{C}, \leq)$.

**State transitions:**
- **Insert:** $\Sigma \mapsto \Sigma \cup \{f\}$ adds a new fact.
- **Delete:** $\Sigma \mapsto \Sigma \setminus \{f\}$ removes a fact.
- **Update:** $\Sigma \mapsto (\Sigma \setminus \{f\}) \cup \{f'\}$ replaces a fact.

**Consistency:** A knowledge state is **consistent** if the sheaf axioms (locality and gluing) hold for all compatible families of sections.

## Definition 13: Canonical Query

A **canonical query** $Q$ is a declarative specification of a set of facts to retrieve, independent of any particular storage engine.

Formally, a query is a tuple:

\[
Q = (\tau, s, r, \vec{o}, c, k_{\min}, n_{\max})
\]

where:

| Component | Symbol | Type | Meaning |
|-----------|--------|------|---------|
| Type | $\tau$ | $\{\text{FACT}, \text{WALK}, \text{JOIN}, \text{GLUE}\}$ | Query mode |
| Subject | $s$ | $\mathcal{E} \cup \{\ast\}$ | Entity to match, or wildcard |
| Relation | $r$ | $\mathcal{R} \cup \{\ast\}$ | Relation to match, or wildcard |
| Objects | $\vec{o}$ | $\mathcal{V}^* \cup \{\ast\}$ | Object pattern, or wildcard |
| Context | $c$ | $\mathcal{C} \cup \{\ast\}$ | Context scope, or wildcard |
| Min confidence | $k_{\min}$ | $[0,1]$ | Minimum confidence threshold |
| Max results | $n_{\max}$ | $\mathbb{N}$ | Limit on returned facts |

**Query denotation:** $\llbracket Q \rrbracket_\Sigma$ is the set of facts in knowledge state $\Sigma$ that match $Q$.

**Implementation:** See `sfdb.query.language.Query`, `common.interfaces.Query`.

---

## Definition 14: Canonical Result

A **canonical result** is the output of executing a canonical query $Q$ against a knowledge state $\Sigma$:

\[
R = \llbracket Q \rrbracket_\Sigma = \{f \in \mathcal{F}_\Sigma \mid f \text{ matches } Q\}
\]

where $\mathcal{F}_\Sigma$ is the set of all facts in $\Sigma$.

**Canonical form:** The result is presented as a set of canonical facts, each consisting of:
- A canonical entity $\varepsilon \in \operatorname{CanonicalEntity}$.
- A canonical relation $\rho \in \operatorname{CanonicalRelation}$.
- An ordered list of values $\vec{\omega}$.
- A context $c$.

**Equivalence:** Two query results $R_1$ and $R_2$ are equivalent iff:
1. $|R_1| = |R_2|$ (same cardinality).
2. There exists a bijection $\phi: R_1 \to R_2$ preserving all components (subject, relation, objects, context, confidence) up to identifier isomorphism.

**Implementation:** See `sfdb.canonical.canonical.CanonicalFact`, `CanonicalModel`.
